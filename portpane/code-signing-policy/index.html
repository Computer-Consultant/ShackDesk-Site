PortPane Code Signing Policy

This page describes how official ShackDesk PortPane builds are produced, signed, and published.

Source Code and Official Builds

PortPane source code is licensed under MIT and is published in the public GitHub repository.

Official ShackDesk builds, optional registration/personalization, and support are separate offerings. These services do not reduce or override MIT rights for the source code.

View the PortPane repository

Current Status

PortPane release artifacts are currently unsigned unless a release explicitly states otherwise.

ShackDesk's initial signing rollout is planned for alpha releases first, then beta. Stable releases may later use the same process or a separate signing process.

Build and Release Process

Official PortPane builds are produced from the public GitHub repository using GitHub Actions workflow runs.

Release artifacts are published to GitHub Releases together with SHA-256 hashes.

When code signing is enabled for a channel, release notes should identify which artifacts were signed and where users can verify current signing status.

Project Roles

Project owner and release maintainer: Mark McDow (N4TEK), My Computer Guru LLC.

Source code, release workflows, and public documentation are maintained in the PortPane GitHub repository.
